# GLOSSARY — единый словарь терминов

> Ubiquitous Language проекта. Эти термины обязательны в коде (английские имена), документации, схемах БД и разговорах. Обнаружил синоним в коде — приведи к термину словаря.

| Термин (RU) | Термин в коде | Определение |
|---|---|---|
| Тенант | `Tenant` | Один отель (или сеть отелей — уточняется в Фазе 4), подключённый к платформе. Единица изоляции данных и конфигурации. |
| Гость | `Guest` | Человек, проживающий или проживавший в отеле. Идентифицируется в рамках тенанта. |
| Проживание | `Stay` | Период пребывания гостя: заезд, выезд, номер, тариф. Связывает гостя с бронью PMS. |
| Сотрудник | `User` | Работник отеля или платформы с ролью (RBAC). Гость — НЕ User. |
| Роль | `Role` | Набор прав: receptionist, housekeeping, engineering, it, fnb, manager, director, marketing, accountant, platform_admin. |
| Заявка | `ServiceRequest` | Единица работы для службы отеля: уборка, ремонт, IT, F&B, жалоба. Один жизненный цикл для всех категорий. |
| Категория заявки | `RequestCategory` | Конфигурируемый тип заявки: маршрутизация в службу, SLA, поля. |
| Диалог | `Conversation` | Переписка гостя (или сотрудника) с платформой в одном канале. |
| Канал | `Channel` | Способ общения: telegram, whatsapp, email, web. |
| Оркестратор | `Orchestrator` | Компонент `ai/`, управляющий диалогом: контекст → решение → ответ/инструмент/эскалация. |
| Инструмент | `Tool` | Декларативная способность AI: типизированные вход/выход, класс подтверждения, права. Обёртка над сервисом ядра. |
| Класс подтверждения | `ConfirmationClass` | `auto` / `confirm_guest` / `confirm_staff` — кто должен подтвердить действие (P-9). |
| База знаний | `KnowledgeBase` | Версионируемые документы тенанта (отель) и общие (город) для RAG. |
| Порт | `Port` | Интерфейс внешней возможности, объявленный ядром (PmsPort, TaxiPort…). |
| Адаптер | `Adapter` | Реализация порта для конкретного провайдера (OperaAdapter, KaspiAdapter…). Живёт в `integrations/`. |
| Эскалация | `Escalation` | Передача диалога от AI живому сотруднику. |
| Реквизиты компании | `CompanyProfile` | Юридические данные корпоративного клиента (БИН, ИИК, адрес…), извлечённые из документов и подтверждённые человеком. |
| АВР | `WorkCompletionAct` | Акт выполненных работ — финансовый документ для корпоративных клиентов (стандарт РК). Выпускается только после подтверждения бухгалтером. |
| Доменное событие | `DomainEvent` | Факт, опубликованный модулем (`request.created`). Доставляется через outbox. |
| Онбординг тенанта | `TenantOnboarding` | Процесс подключения нового отеля: профиль → знания → интеграции → персонал → чек-лист. |
| Каталог ошибок | Error Catalog | Реестр кодов ожидаемых ошибок (`ERR-<MODULE>-NNN`) со статьями диагностики в runbooks. |
| Golden set | Golden Set | Эталонный набор диалоговых сценариев для регрессионной проверки качества AI (evals). |
| Идентификатор гостя | `GuestIdentity` | Идентификатор гостя в конкретном канале: телефон, telegram_id, email, профиль PMS. Один `Guest` — много идентификаторов; модель поддерживает слияние дубликатов. |
| Факт о госте | `GuestFact` | Структурированная запись памяти о госте (предпочтение, аллергия, привычка) с источником и сроком жизни. Хранится отдельно от базы знаний и RAG. |
| AI-шлюз | `AiGateway` | Единственный компонент, через который проходят все вызовы LLM: маршрутизация моделей, лимиты, приоритеты, логирование стоимости и версий промптов. |
