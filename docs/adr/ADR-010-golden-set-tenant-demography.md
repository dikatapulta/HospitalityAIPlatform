# ADR-010: Golden-set покрывает демографию тенантов, а не фиксированные 3 языка

- **Статус:** accepted
- **Дата:** 2026-07-13
- **Автор:** LLM-сессия Task 0015

> Правит формулировку FOUNDATION §7.7 через процесс (R-11): раздел 7.7
> называет golden-set «на 3 языках», но число языков — свойство демографии
> тенанта, а не универсальная константа. Изменение конституции — только
> осознанно, через ADR (FOUNDATION §0), поэтому не правка «попутно».

## Контекст

§7.7 требует golden-set «на 3 языках». Демография первого пилота (решено в
`docs/DISCUSSION_LOG.md`, тема выбора модели): ~100 заселений/день, ~70%
иностранцы (английский, китайский, турецкий, хинди), ~30% местные (казахский,
русский) — шесть языков реального трафика, а не три. Инвариант качества §7.4
(галлюцинация цены/правила отеля — дефект P1) не имеет оговорки на язык:
всё, что видит гость, проходит один и тот же порог. Значит, «3 языка» в §7.7
недостаточно покрывают то, что реально увидит гость пилота, а низкоресурсный
казахский — спящий риск, который сильнее просаживается на дешёвой модели.

Проблема шире одного пилота: у следующего тенанта демография будет иной.
Зашивать в конституцию любое конкретное число языков — заведомо неверно.

Ограничения: §7.4 (P1-бар галлюцинаций без оговорки на язык/тариф), §7.7
(evals перед деплоем промптов/моделей/RAG), P-11 (различия тенантов — данные,
а не форки), P-1 (не проектировать под гипотетику).

## Рассмотренные варианты

1. **Оставить «3 языка», а 6 языков пилота держать только в спеке 0015.**
   Не трогает FOUNDATION, но конституция продолжает называть 3 языка целью;
   будущая LLM-сессия, читающая только §7.7, недопокроет демографию тенанта —
   мёртвая формулировка хуже отсутствующей.
2. **Зашить «6 языков» в §7.7.** Верно для этого пилота, неверно для
   следующего тенанта с другой демографией; снова придётся править конституцию.
3. **Обобщить правило: golden-set покрывает языки реальной демографии
   тенантов; конкретный набор — свойство тенанта, а не константа §7.7.**
   Правило перестаёт устаревать при смене состава тенантов.

## Решение

**Вариант 3.** §7.7 обобщается: golden-set покрывает языки реальной демографии
обслуживаемых тенантов, а не фиксированное число. Конкретный набор — свойство
тенанта (P-11: данные, не код). Для первого пилота набор — **6 языков**:
английский, русский, казахский, китайский, турецкий, хинди (en/ru/kk/zh/tr/hi).
Высокоставочный контент (цены/правила/бронь — P1-бар §7.4) проверяется на всех
языках набора; приоритет проверки — казахский (низкоресурсный, спящий риск) и
корректность перевода знаний RAG, авторящихся на ru/kk/en, в zh/tr/hi на лету.
Высокоставочные действия (деньги, документы, изменение брони) и так под
подтверждением человека (P-9/NG-4).

## Последствия

- FOUNDATION §7.7 переформулирован: «на 3 языках» → «на языках реальной
  демографии тенантов (ADR-010; для первого пилота — 6: en/ru/kk/zh/tr/hi)».
  Версия документа в шапке помечена.
- Спека 0015 конкретизирует 6-язычный golden-set пилота и bake-off Haiku 4.5 vs
  Sonnet 5 на нём (§7.7): выбор рантайм-модели — по данным, не по интуиции.
- Fidelity-проверка перевода знаний RAG в полном объёме требует RAG (Phase 1);
  в Phase 0 (нет источника знаний) 6-язычный набор проверяет выбор инструмента,
  подтверждение и отказ-от-галлюцинации/эскалацию — задел на Phase 1 помечен.
- Пересмотреть: при подключении тенанта с иной языковой демографией набор
  языков его golden-set пересматривается вместе с онбордингом — правка §7.7 не
  требуется (в этом и смысл обобщения).
