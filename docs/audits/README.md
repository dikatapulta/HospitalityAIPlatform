# Аудиты фазы

Независимые аудиты, проводимые отдельными LLM-сессиями с чистым контекстом
(правило исполнения 8 в `PROJECT_EXECUTION_PLAN.md`: Exit Criteria каждой фазы
включают такой аудит). Отчёты сюда кладутся как есть — это **запись состояния на
дату**, а не живой документ: их не переписывают задним числом, найденное
превращают в issue/ADR (R-11).

## Именование

`ГГГГ-ММ-ДД-<тема>-audit.md` — дата проведения, не дата коммита.

## Историческая сноска про путь

Операционный аудит изначально был записан в `docs/audit/OPERATIONAL_AUDIT.md`
(каталог в единственном числе — так просил его промпт). Каталог унифицирован в
`docs/audits/`, файл живёт как `2026-07-21-operational-audit.md`. Ссылки на
старый путь встречаются внутри отчётов и промптов — они оставлены нетронутыми
намеренно (запись того, что видел аудитор), читать их следует через эту сноску.

## `prompts/`

Промпты, которыми запускались аудиты. Хранятся ради воспроизводимости: правило 8
требует такой аудит на выходе **каждой** фазы, и следующий прогон начинается с
копии соответствующего промпта, а не с чистого листа. Это тоже исторические
записи — правится копия под новую фазу, а не архив.
