# Промпт: оптимизационный аудит (параллельное окно 1/3)

Ты — аудитор кода с чистым контекстом. Проведи оптимизационный аудит проекта: лишний код, неэффективные и переусложнённые решения, кандидаты на упрощение.

**Направление.** Мёртвый/недостижимый код и неиспользуемые абстракции; дублирование против канонов; неэффективные запросы к БД (N+1, отсутствующие индексы, лишние roundtrip'ы); избыточные слои там, где FOUNDATION требует простоты («один канонический паттерн», файл ≤400 строк); зависимости — прогнать `pip-audit`, найти неиспользуемые и устаревшие пакеты. Помни контекст: код пишут LLM, сопровождаемость важнее элегантности — «умный» код тоже находка.

**Не дублируй** находки существующих аудитов: `docs/audits/2026-07-21-security-audit.md`, `docs/audit/OPERATIONAL_AUDIT.md`, `docs/audits/2026-07-22-legal-audit-rk.md`. Если находка смежная — сошлись на неё одной строкой.

**Правила изоляции (параллельно работают другие сессии):**
- Только чтение кода. Никаких правок, коммитов, `git checkout/branch/stash`.
- Не запускать приложение, docker, `make check`. Разрешены read-only команды (grep, pip-audit, mypy --no-error-summary и т.п.).
- Чужие untracked-файлы в `git status` — норма, не трогать.
- Писать ровно один файл: `docs/audits/2026-07-23-optimization-audit.md`.

**Формат отчёта:** вердикт → находки по убыванию ценности (каждая: суть, файл:строка, оценка выигрыша — строки/токены/мс, риск правки) → чего НЕ делать (где упрощение навредит) → предлагаемая упаковка в задачи.
