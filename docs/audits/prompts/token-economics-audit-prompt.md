# Промпт: аудит токен-экономики (параллельное окно 2/3)

Ты — аудитор с чистым контекстом. Посчитай экономику LLM-использования и найди, где платформа переплачивает за токены.

**Направление.** Стоимость одного полного диалога гостя по составляющим: оркестратор (`ai/orchestrator.py` — system-промпт + история до 20 сообщений), переводы для персонала и гостя (`ai/translation.py`), повторные вызовы. Фактические данные — таблица `llm_call_log` на staging (read-only SQL по ssh; доступ описан в `docs/runbooks/`, хост в локальном `.env`). Экстраполяция на профиль пилота из ADR-010 (~100 заселений/день, 70% иностранцы). Кандидаты на экономию: prompt caching (system-промпт и история — идеальные кандидаты), где Haiku безопасен вместо Sonnet (учти память проекта: Haiku на казахском сползает в русский — фикс #71/#72), длина system-промпта, окно истории, дедупликация переводов. Сверь с бюджетом: самофинансируемый пилот ~$100–200, дневной кап тенанта ~$5.

**Итог должен отвечать на вопрос:** хватит ли $100–200 на месяц пилота, и какие 2–3 правки дают наибольшую экономию без потери качества.

**Правила изоляции (параллельно работают другие сессии):**
- Только чтение кода и read-only SQL на staging. Никаких правок, коммитов, изменений на сервере.
- Не запускать приложение/docker/make check локально.
- Чужие untracked-файлы в `git status` — норма, не трогать.
- Писать ровно один файл: `docs/audits/2026-07-23-token-economics-audit.md`.

**Формат отчёта:** вердикт (стоимость диалога / прогноз месяца пилота) → разбивка затрат → находки-экономии по убыванию эффекта ($ в месяц, риск) → рекомендации по конфигурации моделей.
