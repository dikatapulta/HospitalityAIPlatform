# Чеклист секретов (staging)

> Task 0006. FOUNDATION §11: секретов в репозитории нет — только в секрет-хранилище
> (GitHub Secrets) и в переменных окружения на сервере (`.env`). Этот чеклист —
> единый список: что за секрет, где живёт, как ротировать. Обновляется при
> появлении каждого нового секрета (интеграции, LLM-ключи — в следующих задачах).

## 1. GitHub Actions Secrets (репозиторий → Settings → Secrets and variables → Actions)

Нужны job'у `deploy-staging` (см. [deploy.md](deploy.md)).

| Секрет | Что это | Обязателен | Ротация |
|---|---|---|---|
| `STAGING_SSH_HOST` | IP/хост VPS staging. Пока пуст — деплой пропускается. | да | при смене сервера |
| `STAGING_SSH_USER` | Пользователь деплоя на VPS (`deploy`). | да | редко |
| `STAGING_SSH_KEY` | **Приватный** SSH-ключ для деплоя (отдельная пара, только для CI). | да | при подозрении на утечку — перегенерировать (A3 в deploy.md), заменить `authorized_keys` и секрет |
| `STAGING_SSH_PORT` | SSH-порт, если не 22. | нет | — |

`GITHUB_TOKEN` — встроенный, отдельно заводить не нужно: им job логинится в GHCR
для push образа (`packages: write` в workflow).

## 2. Секреты на сервере (`/opt/hospitality/.env`, шаблон — [ops/deploy/.env.staging.example](../../ops/deploy/.env.staging.example))

Живут **только** на VPS, не в репозитории и не в GitHub Secrets.

| Переменная | Что это | Ротация |
|---|---|---|
| `POSTGRES_PASSWORD` | Пароль БД staging. Сгенерировать: `openssl rand -hex 24`. | сменить в `.env` → `deploy.sh` пересоздаст контейнер app с новым паролем; для существующей БД пароль менять и в Postgres |
| `POSTGRES_USER`, `POSTGRES_DB` | Имя пользователя/БД. | редко |
| `APP_PORT` | Порт приложения на VPS (по умолчанию 8000). | — |

## Правила
- Ничего из этого **никогда** не коммитится: `.env` — в `.gitignore`, приватные ключи — только в GitHub Secrets и на сервере.
- Ключ деплоя — **отдельная** пара, не личный ключ разработчика; компрометация → перегенерация, а не общий доступ.
- Утечка любого секрета → ротация по таблице выше + запись в лог инцидентов.
- Новый секрет в проекте (LLM-ключ, токен Telegram, ключ интеграции) → строка в этом чеклисте в том же PR.
