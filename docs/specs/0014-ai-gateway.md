# Спека 0014 — AI Gateway: единственная дверь к LLM

- **Задача:** Task 0014 (PHASE0.md), FOUNDATION §7.2, R-5.
- **Статус:** реализована в этом же PR (спека и код ревьюятся вместе — фаза 0).

## Проблема

Платформе предстоит первый AI-вызов (оркестратор, Task 0015). До него должно
быть физически верно «ни один токен мимо gateway» (§7.2): единая точка
таймаутов/ретраев, учёта стоимости и бюджета тенанта, логирования каждого
вызова. Прямые вызовы провайдера из любого другого места — запрещены и
отлавливаются import-linter'ом.

## Решение

Пакет `src/hospitality/ai/gateway/` по анатомии канонического модуля
(`modules/requests`): вход только через `api.py`.

| Файл | Что даёт |
| --- | --- |
| `schemas.py` | Pydantic-границы: `LlmMessage`, `LlmRequest`, `LlmResponse` (R-6) |
| `provider.py` | Порт `LlmProvider` (Protocol) + результат `LlmProviderResult` + ошибки порта (timeout / provider error) |
| `anthropic_provider.py` | Адаптер Anthropic: одна модель из настроек, SDK-ретраи выключены (ретраи — забота gateway) |
| `mock_provider.py` | Fake-адаптер порта (ADR-007): рождается в одном PR с портом; настраиваемые токены/таймауты, копит вызовы |
| `models.py` | `LlmCallLog` — тенантная таблица журнала вызовов (канон RLS) |
| `service.py` | `complete()`: бюджет → ретраи по таймауту → расчёт стоимости → запись в `LlmCallLog` + structlog |
| `api.py` | Публичный интерфейс gateway |

### Контракт вызова

`complete(LlmRequest, provider=...) -> LlmResponse` вызывается внутри
`tenant_context(...)` (P-4). Порядок: проверка дневного бюджета тенанта →
до `LLM_MAX_ATTEMPTS` попыток вызова провайдера (ретрай только по таймауту) →
расчёт стоимости по прайс-листу модели → запись `LlmCallLog` (тенант,
correlation id, хэш промпта sha256, токены, стоимость, latency, статус) →
событие `llm_call` в логах. Исход каждого вызова — включая таймаут и ошибку
провайдера — виден в БД (DoD Task 0014).

- Маршрутизации моделей нет (Non-Goal задачи): одна модель `LLM_MODEL`.
- Бюджет — простейший: `LLM_TENANT_DAILY_BUDGET_USD` одинаков для всех
  тенантов; сумма `cost_usd` за текущие UTC-сутки ≥ бюджета → отказ до
  обращения к провайдеру. Пер-тенантный бюджет — конфиг тенанта, Phase 1.
- Стоимость считается по модели, сконфигурированной у адаптера
  (детерминированно), из прайс-листа `MODEL_PRICING_USD_PER_MTOK`.

### Ошибки (docs/runbooks/errors.md)

| Код | Когда | HTTP |
| --- | --- | --- |
| ERR-AI-001 | таймауты провайдера исчерпали ретраи | 503 |
| ERR-AI-002 | дневной бюджет тенанта исчерпан | 429 |
| ERR-AI-003 | ошибка провайдера (не таймаут) | 502 |

### Import-linter (R-5)

Контракт 4 «LLM providers only via ai/gateway» включается по-настоящему:
в `source_modules` добавляются `hospitality.app`, `hospitality.worker`,
`hospitality.tools`; `allow_indirect_imports = true` — запрещён именно
*прямой* импорт SDK (транзитивный путь composition root → orchestrator →
gateway → anthropic легален и неизбежен). Проверено нарочным нарушением
(результат — в описании PR). Попутно `hospitality.tools` добавлен в
контракт 5 (рекомендация ревью PR #26).

## План тестов (R-7)

На mock-провайдере: вызов логируется в БД со стоимостью, хэшем промпта и
correlation id; ретрай при таймауте (успех со второй попытки; исчерпание →
ERR-AI-001 и строка `timeout` в журнале); бюджетный лимит (превышение →
ERR-AI-002 без обращения к провайдеру; бюджет соседнего тенанта не тратится).
Контрактный тест анфропик-адаптера — на заглушке SDK-клиента (перевод
ответа/таймаута в типы порта). Нарочное нарушение линтера — вручную,
зафиксировано в PR.
