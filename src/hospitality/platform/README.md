# platform — тенанты, пользователи, конфигурация (kernel)

Назначение: корневой модуль платформы — реестр тенантов, а в следующих задачах
пользователи, RBAC, аудит, конфигурация и фиче-флаги (FOUNDATION §5.1).
Полный паспорт модуля появится в Task 0011 вместе с конфигом тенанта.

| --- | --- | --- |

## Таблицы

- `tenants` (миграция `0001`) — реестр тенантов: `id` (UUID), `slug`
  (уникальный человекочитаемый идентификатор), `name`, `created_at`,
  `updated_at`. Таблица НЕ тенантная (это сам реестр), поэтому без
  `tenant_id`/RLS; RLS-канон для тенантных таблиц — Task 0009.
